Start Over

This abstraction clears the play area of all graphical effects (removes all drawing that has occurred) puts the cake object outside the observable screen, and sets the size of the sprite to twenty-five per cent of that of its regular size, every time the 'start' button is pressed.

Shake Present

On start, this function sets the object so it aligns it towards the top of the screen, then it shakes the object right 15 degrees, left 30 degrees, then right 15 degrees, until the variable, not clicked, equals zero.

Fade In

This Abstraction fades in a completely transparent object by increasing it's opacity by 10% ten times until the object is fully visible.

All Loops SONG1

This function calls all of the loops for a song, three of which repeats a beat each, and the final loop alternates two sound files for the entire song.

All Noises

This Abstraction holds two loops, that each alternate two different sound files, the first loop altenating s3piano3 and s3piano4, and the second alternating the files s3piano4 and s3piano6

Conditionals

This abstraction calls itself at the start of the program, and processes the input of the user each time it is allowed to be inputted. Based on the input, checked with a list of titles, the program either calls the functions Song1, Song2, or Song3. If the input is 'random' then the program picks a random number (1-3) and chooses a song based on that. If the input given doesn't match any known inputs foe the program, then the program tells the user that their input is invalid.

Titlescreen

This function calls this text- art each time the program starts, or the game is restarted after the user's health equals zero.

Market Action

This algorithm tests a user's input and then disserns if the user has a high enough playermoney value to increase their player attack or player defense by the value of the price of the equipment. If so, then the item level for that object is increased by one, money is taken away, and the desired variable is increased. If not, then an error message is displayed and no 'transaction' occurs.

Start Input

This Abstraction is called when the program starts, or when it restarts after the palyer 'dies' and it checks a user's input. If the input equals 'start' then the program continues and the start() function is called, but if the user doesn't input 'start' then an error message is displayed and the user is asked for input again until they input 'start'.

Display

This abstraction prints out a user's 'stats' eachntime a new event occurs in the program by printing a discription of a variable and the dalue of the variable itself.
